# Сервис синхронизации файлов

Сервис для автоматической синхронизации файлов между локальной папкой на компьютере пользователя и облачным хранилищем **Яндекс Диск**.

Программа отслеживает изменения в указанной директории и автоматически приводит облачное хранилище в соответствие: новые и изменённые файлы загружаются в облако, удалённые локально — удаляются из облака.

## Возможности

- **Автоматическая синхронизация** — сервис работает в фоновом режиме с заданным интервалом
- **Однонаправленная синхронизация** — локальная папка является источником данных, облако приводится в соответствие с ней
- **Обработка ошибок** — сетевые сбои и ошибки чтения файлов не прерывают работу программы
- **Логирование** — все операции фиксируются в файл лога с уровнем INFO или ERROR
- **Валидация при запуске** — проверяются наличие папки, валидность токена и обязательные параметры конфигурации
- **Расширяемость** — логика работы с облачным хранилищем изолирована в отдельном классе-коннекторе, что позволяет добавить поддержку других провайдеров

## Требования

- Python 3.10+
- Операционная система: Windows / Linux / macOS
- Аккаунт Яндекс Диска с созданным OAuth-токеном

## Установка

### 1. Клонирование репозитория

```bash
git clone <url-репозитория>
cd file_synchronization_service
```

### 2. Создание виртуального окружения

**Linux / macOS:**

```bash
python -m venv .venv
source .venv/bin/activate
```

**Windows:**

```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 4. Получение токена Яндекс Диска

1. Перейдите на страницу https://oauth.yandex.ru/authorize?response_type=token&client_id=ClientID.
Вместо ClientID подставьте значение идентификатора приложения.
2. Скопируйте полученный OAuth-токен и сохрание его

## Настройка

Создайте файл `.env` в корне проекта на основе `.env.example`:

```bash
cp .env.example .env
```

Заполните параметры:

| Параметр           | Описание                                              | Пример                          |
|--------------------|-------------------------------------------------------|---------------------------------|
| `SYNC_FOLDER_PATH` | Абсолютный путь к локальной папке для синхронизации   | `/home/user/sync_folder`        |
| `CLOUD_FOLDER_NAME`| Имя папки на Яндекс Диске для хранения бэкапов        | `backup`                        |
| `YANDEX_TOKEN`     | OAuth-токен доступа к Яндекс Диску                    | `y0_AgAAAABz...`                |
| `SYNC_INTERVAL`    | Интервал синхронизации в секундах                     | `10`                            |
| `LOG_FILE_PATH`    | Абсолютный путь к файлу лога                          | `/home/user/sync.log`           |

> **Важно:** файл `.env` добавлен в `.gitignore` и не должен попадать в репозиторий.

Пример содержимого `.env`:

```env
SYNC_FOLDER_PATH=/home/user/sync_folder
CLOUD_FOLDER_NAME=backup
YANDEX_TOKEN=y0_AgAAAABz...
SYNC_INTERVAL=10
LOG_FILE_PATH=/home/user/sync.log
```

## Запуск

```bash
python main.py
```

После запуска:

1. Программа прочитает конфигурацию из `.env` и проверит все параметры
2. Выполнит первую синхронизацию (содержимое облака приведётся в соответствие с локальной папкой)
3. Перейдёт в бесконечный цикл синхронизации с заданным интервалом
4. Для завершения нажмите `Ctrl+C` — программа корректно остановится и запишет лог

## Логирование

Логи записываются в файл, указанный в параметре `LOG_FILE_PATH`.

**Формат записи:**

```
2025-01-15 14:30:00 | INFO | Запуск приложения
2025-01-15 14:30:00 | INFO | Синхронизируемая папка: /home/user/sync_folder
2025-01-15 14:30:01 | INFO | Загружен в облако: document.txt
2025-01-15 14:30:05 | INFO | Перезаписан в облаке: notes.md
2025-01-15 14:30:08 | INFO | Удалён из облака: old_file.txt
2025-01-15 14:30:12 | ERROR | Ошибка загрузки /home/user/sync_folder/big.dat: ...
2025-01-15 14:35:00 | INFO | Завершение работы приложения
```

**Параметры логгера:**

- Уровень логирования: `INFO`
- Ротация: при достижении файла размера 10 МБ создаётся новый файл
- Кодировка: UTF-8
